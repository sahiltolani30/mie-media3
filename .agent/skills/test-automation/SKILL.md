name: test-automation
description: Automates the creation, updating, and execution of unit and integration tests.
---

# Instructions
- Identify the tech stack in the `frontend` and `backend` (e.g., Jest, Pytest, Mocha).
- When new features are implemented, proactively suggest or write corresponding unit tests.
- If existing code is modified, check if existing tests need updates.
- Ensure test coverage for edge cases and error paths.
- Run tests using `run_command` and analyze output for failures.
- Provide clear summaries of test results and fix any bugs identified by test failures.
